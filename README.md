# AMR Line Tracing Robot

STM32 기반으로 제작한 자율주행 주행로봇 프로젝트입니다.  
라인트레이싱을 기본으로 하고, 전방/측면 초음파 센서를 이용해 장애물을 감지하면 감속 또는 정지하도록 구현했습니다.

## 1. 프로젝트 개요

이 프로젝트는 검은 선을 따라 주행하는 AMR 형태의 소형 로봇을 제작하고,  
주행 중 장애물을 감지했을 때 안전하게 반응하도록 제어 로직을 구성하는 것을 목표로 했습니다.

단순히 센서값에 바로 반응하는 방식보다,  
주행 상태와 장애물 상태를 나누어 판단하면서 조금 더 안정적으로 동작하도록 만드는 데 중점을 두었습니다.

## 2. 개발 환경

- MCU : STM32F411RE
- Language : C
- IDE : STM32CubeIDE
- Motor Driver : BTS7960
- Sensor : 라인센서 2개, 초음파센서 3개
- Debug : UART 출력

## 3. 주요 기능

- 라인센서 2개를 이용한 라인트레이싱
- 전방 초음파 센서를 이용한 장애물 거리 판단
- 좌/우 초음파 센서를 이용한 측면 장애물 감지
- 장애물 거리 조건에 따른 주행 / 감속 / 정지
- 버저를 이용한 경고 출력
- UART를 통한 센서값 및 상태 확인

## 4. 동작 방식

### 라인트레이싱
좌측, 우측 라인센서 값을 읽어서 검은 선의 위치를 판단했습니다.

- 두 센서가 모두 흰색이면 직진
- 왼쪽만 검은색이면 우회전 방향 보정
- 오른쪽만 검은색이면 좌회전 방향 보정
- 두 센서가 모두 검은색이면 최근 주행 방향을 참고해 회전 방향 결정

센서가 한 번만 반응했을 때 바로 급하게 꺾이지 않도록,  
연속 감지 횟수를 기준으로 완만한 회전과 강한 회전을 나누었습니다.

### 장애물 판단
초음파 센서로 전방과 좌우 거리를 측정한 뒤,  
전방 거리 기준으로 주행 상태를 나누었습니다.

- 일정 거리 이상이면 정상 주행
- 가까워지면 감속
- 매우 가까우면 정지

좌우 측면에도 가까운 물체가 있으면 버저를 울리도록 했습니다.

## 5. 구현하면서 중점적으로 본 부분

처음에는 센서값이 순간적으로 바뀔 때 조향이 너무 급하게 바뀌는 문제가 있었습니다.  
그래서 한 번의 입력으로 바로 방향을 크게 바꾸기보다,  
연속 검출 횟수와 속도 차이를 이용해서 보정 강도를 나누는 방식으로 수정했습니다.

또한 장애물 감지 기능을 추가하면서,  
라인 추종 로직과 장애물 정지 로직이 서로 섞이지 않도록  
주행 판단과 모터 출력 단계를 분리해서 정리했습니다.

## 6. 맡은 역할

- STM32 기반 전체 주행 로직 작성
- 라인센서 입력 처리
- 초음파 센서 거리 측정 로직 구현
- 장애물 감속/정지 조건 설정
- 모터 PWM 제어
- UART 디버깅
- 주행 테스트 및 파라미터 조정

## 7. 아쉬웠던 점

초음파 센서 측정은 busy-wait 방식으로 구현되어 있어서  
구조적으로는 더 개선할 여지가 있다고 생각했습니다.  
이후에는 RTOS 기반으로 센서, 판단, 출력 기능을 분리하는 방향으로도 정리했습니다.

## 8. 실행 결과

- 라인을 따라 기본 주행 가능
- 전방 장애물 접근 시 감속 또는 정지 가능
- 측면 장애물 접근 시 버저 경고 가능
- UART로 센서값과 상태 확인 가능

## 9. 파일 구성

- `main.c` : 전체 제어 로직
- `motor control` 관련 함수 : 모터 PWM 및 방향 제어
- `ultrasonic` 관련 함수 : 거리 측정
- `line sensor` 관련 함수 : 라인 감지
- `uart debug` 관련 코드 : 상태 출력
